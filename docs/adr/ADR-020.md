# ADR-020: FIFO Challenge Matching

**Date:** 2026-06-14  
**Status:** ACCEPTED  
**Story:** STORY-38  

---

## Context

Social Prediction marketplace challenges allow multiple fans to accept the same listing. The order of acceptance needs to be deterministic, fair, and auditable.

## Decision

Challenge acceptance uses First-In-First-Out (FIFO) order based on `SocialPredictionMatch.createdAt`. No randomness, no weighting by Fan Value or any other signal.

Partial matches are allowed — a fan can accept only part of a listing's capacity. Full matches fill all remaining capacity.

## Alternatives Considered

- **Random matching** — rejected; not auditable and perceived as unfair
- **Highest Fan Value gets priority** — rejected; creates pay-to-win dynamics and requires Fan Value query on every match
- **Dutch auction / price discovery** — rejected; introduces financial complexity inconsistent with gameplay points

## Consequences

- Deterministic ordering — no hidden weighting
- First to accept gets first fill on a limited-capacity listing
- Auditable — `ChallengeMatch` records show order via `matchedAt`
- Idempotency key prevents a fan from accepting the same listing twice (ADR-021)
- No priority queue or weighted matching algorithms needed
