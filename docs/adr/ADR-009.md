# ADR-009: Testing Strategy

**Date:** 2026-06-08
**Status:** ACCEPTED
**Owner:** All agents
**Reviewed By:** Programme Director Agent, Technical Review Board

## Context

PSL One handles financial transactions (loyalty points), competitive game integrity (fantasy scoring, GTS settlement), and personal data (POPIA). Bugs in these domains cause real harm: incorrect scores, fraudulent point awards, data breaches. The testing strategy must be rigorous enough to catch regressions across 15 services while remaining fast enough for a 12-week build cadence.

CLAUDE.md mandates: "Always write tests."

Release criteria: >80% line coverage on every service. CI blocks merge on coverage failure.

## Decision

| Area | Tool | Rationale |
|---|---|---|
| Unit tests | **Vitest** | Faster than Jest, native ESM, excellent TypeScript support, compatible with NestJS |
| Integration tests | **Supertest + Testcontainers** | Real HTTP + real database (no mocks) |
| E2E tests | **Playwright** | Cross-browser, excellent for Next.js, component-level and full-page |
| Contract tests | **Pact.js** | Consumer-driven contract testing for Kafka event schemas |
| Load tests | **K6** | Scripted load tests, GitHub Actions integration |
| Coverage | **V8 coverage (Vitest built-in)** | Native, no instrumentation overhead |
| Minimum coverage threshold | **80% line coverage** | Enforced by CI — PR blocked below threshold |

## Options Considered

### Unit test framework
- Vitest: native ESM, faster than Jest, vitest ui, compatible with NestJS
- Jest: established, more plugins — rejected for slower startup and ESM compatibility friction

### E2E framework
- Playwright: cross-browser, excellent DX, first-class Next.js support — CHOSEN
- Cypress: limited multi-browser support, slower — rejected

### Contract testing
- Pact.js: consumer-driven contracts for both HTTP and message (Kafka) — CHOSEN
- No contract testing: rejected — without contracts, Kafka schema changes silently break consumers

### Integration testing
- Testcontainers (real PostgreSQL + Redis): no mock/real divergence — CHOSEN
- Mock database: rejected — "got burned last quarter when mocked tests passed but prod migration failed" pattern risk

## Consequences

### Positive
- 80% coverage threshold enforced in CI means no service ships with significant uncovered code
- Pact contracts catch Kafka schema breakage before it reaches consumers
- Testcontainers integration tests catch real database behaviour (transactions, constraints, deadlocks)
- Playwright E2E covers the user journeys that matter most

### Negative
- Testcontainers requires Docker in CI (GitHub Actions supports this)
- Pact broker needed for contract sharing between producer and consumer services

### Risks
- Slow test suite blocking CI: mitigated by Turborepo affected-only test runs + Vitest parallel workers
- Flaky E2E tests: mitigated by Playwright retry configuration and stable test selectors (data-testid)

## Implementation Notes

**Vitest configuration (per service):**
```typescript
// services/<name>/vitest.config.ts
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    globals: true,
    environment: 'node',
    coverage: {
      provider: 'v8',
      reporter: ['text', 'lcov'],
      thresholds: { lines: 80, branches: 80, functions: 80, statements: 80 },
      exclude: ['**/node_modules/**', '**/dist/**', '**/*.spec.ts', '**/main.ts', '**/*.module.ts'],
    },
  },
});
```

**NestJS test pattern:**
```typescript
// Unit test — pure domain logic, no framework
describe('FantasyScoreCalculator', () => {
  let calculator: FantasyScoreCalculator;
  beforeEach(() => { calculator = new FantasyScoreCalculator(); });
  
  it('awards 6 points for a goalkeeper goal', () => {
    const points = calculator.calculate({ position: 'GK', goals: 1 });
    expect(points).toBe(6);
  });
});

// Integration test — real database via Testcontainers
describe('IdentityService (integration)', () => {
  let app: INestApplication;
  
  beforeAll(async () => {
    // Testcontainers spins up a real PostgreSQL instance
    const moduleRef = await Test.createTestingModule({
      imports: [AppModule],
    }).compile();
    app = moduleRef.createNestApplication();
    await app.init();
  });
  
  it('POST /auth/register creates user and publishes event', async () => {
    const res = await request(app.getHttpServer())
      .post('/auth/register')
      .send({ email: 'test@pslone.co.za', mobile: '+27821234567', password: 'Test1234!' });
    expect(res.status).toBe(201);
    // Verify Kafka event was published (via test Kafka consumer)
  });
});
```

**Pact contract test (Kafka producer):**
```typescript
// services/identity/src/events/user-registered.pact.spec.ts
describe('UserRegistered Kafka contract', () => {
  it('produces an event matching the consumer contract', async () => {
    const event = buildUserRegisteredEvent({ userId: uuid(), email: 'fan@test.com' });
    const parsed = UserRegisteredPayloadSchema.safeParse(event.payload);
    expect(parsed.success).toBe(true);
    // Pact publishes this contract to the Pact Broker for consumer validation
  });
});
```

**K6 load test (match centre spike):**
```javascript
// infra/load-tests/matchday-spike.js
import http from 'k6/http';
export const options = {
  stages: [
    { duration: '2m', target: 1000 },   // ramp up
    { duration: '5m', target: 10000 },  // matchday peak
    { duration: '2m', target: 0 },      // scale down
  ],
  thresholds: {
    http_req_duration: ['p(95)<300'],   // p95 < 300ms
    http_req_failed: ['rate<0.01'],     // < 1% error rate
  },
};
```

**CI pipeline test stages:**
```yaml
# .github/workflows/ci.yml
jobs:
  test:
    steps:
      - run: pnpm turbo run test --filter=...[origin/main]  # affected only
      - run: pnpm turbo run test:e2e --filter=apps/web      # always on web changes
      - run: pnpm turbo run test:contracts                  # Pact contracts
```

**Testing rules (all agents must follow):**
1. Every domain service method has a unit test
2. Every Kafka producer has a Pact contract test
3. Every API endpoint has a Supertest integration test
4. Integration tests use Testcontainers — no mocking the database
5. E2E tests cover the happy path for every must-have user journey
6. Coverage report posted to every PR as a comment
7. CI blocks merge if coverage drops below 80%
8. No `it.skip` or `xit` without a linked GitHub issue
