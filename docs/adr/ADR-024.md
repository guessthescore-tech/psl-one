# ADR-024: Dry Run Response Contract

**Date:** 2026-06-14  
**Status:** ACCEPTED  
**Story:** STORY-36, STORY-39  

---

## Context

Several operations (season activation, rollback, price calibration) are high-stakes. Operators need to understand what *would* happen before committing. A dry-run mode is needed that:
1. Cannot accidentally perform the real operation
2. Clearly signals to callers that no writes occurred
3. Is machine-readable (not just a message string)

## Decision

All dry-run operation responses must include these fields:

```typescript
{
  dryRunOnly: true,                        // always true in a dry-run response
  activationWillNotBePerformed: true,      // for activation dry-runs
  rollbackWillNotBePerformed: true,        // for rollback dry-runs (where applicable)
  worldCupHistoryPreserved: true,          // for rollback dry-runs
  // ... operation-specific description fields
}
```

No writes are performed in dry-run mode. If a dry-run method calls any Prisma `.create()` or `.update()`, it is a bug.

## Alternatives Considered

- **Separate `/dry-run` path prefix** — considered; having a distinct URL makes it harder to confuse with a real operation, but adds routing complexity
- **`mode=dry-run` query parameter** — rejected; query parameters can be stripped by middleware, logging, or caches without the caller noticing
- **Message-only response (`"message": "this would activate the season"`)** — rejected; not machine-readable; frontend cannot reliably distinguish dry-run from real response

## Consequences

- Tests can assert `dryRunOnly: true` to verify no writes happened
- Frontend can display a clear "this was a simulation" banner when `dryRunOnly: true`
- Adding a new dry-run endpoint must follow this contract
- The contract is documented in `BetaLaunchService` and referenced in beta runbooks

## Revisit Triggers

- If dry-run routes need to run Prisma transactions and roll them back (would need `$transaction` + explicit throw — add to this ADR)
- If a third-party tool needs to parse dry-run responses (ensure `dryRunOnly` field remains in all dry-run responses)
